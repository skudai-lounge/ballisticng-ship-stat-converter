About

This tool converts simple frontend ship stats into BallisticNG XML stat values.

The five values you enter, Acceleration, Speed, Handling, Shield, and Firepower, are player facing summary stats. BallisticNG uses many backend XML fields to create the actual ship behavior, so one frontend value may control several XML fields.

How the rules were established

The rules were built from stock ship examples. Frontend stat bars were recorded, then matched against the imported XML stat files for those ships. Fields that moved predictably with a frontend value became formulas or lookup tables. Fields that stayed the same across stock examples became defaults. Fields that affected handling feel but did not follow clean math became presets.

This means the converter is a house conversion system based on observed stock ship patterns. It is designed to create predictable custom ship stats, not to recreate every stock ship exactly.

Example calculations

  • Speed 5. Speed uses the lookup table. Speed 5 outputs Toxic 60.5, Apex 74.0, Halberd 89.5, Spectre 105.0, and Zen 119.5.
  • Acceleration 3. Acceleration uses the lookup table. Acceleration 3 outputs Toxic 1.10, Apex 1.00, Halberd 0.90, Spectre 0.70, and Zen 0.50.
  • Shield 6. Shield uses the lookup table. Shield 6 outputs Shield Effectiveness 0.94. This lookup follows the simple pattern 1.00 - 0.01 × Shield.
  • Handling 4. Grip uses the Handling lookup table, so Handling 4 becomes Grip 5.50. Steer values still use formulas, so Steer Speed = 1.486 + 0.054 × 4 = 1.70.
  • Firepower 7. Weapon Effectiveness uses the Firepower lookup table, so Firepower 7 becomes 1.032.

Stock ship examples

  • Firepower lookup alignment. Wyvern uses Firepower 4 and Weapon Effectiveness 0.984. G Tek uses Firepower 5 and Weapon Effectiveness 1.000. Diavolt uses Firepower 8 and Weapon Effectiveness 1.048. This supports using a lookup table instead of one exact formula, because the stock values are close but not perfectly linear.
  • Firepower 7 estimate. Firepower 7 is placed between G Tek at 5 and Diavolt at 8, so the lookup assigns Weapon Effectiveness 1.032 as a controlled higher damage value.
  • Shield formula alignment. Wyvern and Tenrai use Shielding 3 with Shield Effectiveness 0.97. G Tek uses Shielding 5 with Shield Effectiveness 0.95. Diavolt uses Shielding 8 with Shield Effectiveness 0.92. These examples match Shield Effectiveness = 1.00 - 0.01 × Shielding.
  • Speed lookup alignment. G Tek uses a slow profile with Zen 115. Tenrai uses Speed 5 with Zen 119.5. Nexus uses Speed 9 with Zen 125. This supports using stock ship anchors for speed conversion.
  • Handling preset alignment. Precision Delta and G Tek use Handling 8 with Grip 6.5. Omnicom uses Handling 4 with Grip 5.65. Diavolt uses Handling 1 with Grip 5.0. This supports using Handling as a preset driven feel value.

Rule types

  • Lookup values. Speed, Acceleration, Shield, Firepower, and Grip use lookup tables as the source of truth. For example, Speed 5 becomes Toxic 60.5, Apex 74.0, Halberd 89.5, Spectre 105.0, and Zen 119.5. Firepower 7 becomes Weapon Effectiveness 1.032.
  • Simple formulas. Steer values use formulas because they scale smoothly from Handling. Shield can be explained as 1.00 - 0.01 × Shield, but the converter still uses the Shield lookup table for consistency.
  • Presets. Some stats affect feel rather than one clean number. Handling is treated this way for grip. Handling 4 becomes Grip 5.50, Airbrake Drift Power 0.85, Airbrake Drift Gain 5.4, and Airbrake Drift Falloff 14.
  • Defaults. Some XML fields do not come from frontend stats. Engine Power, Gravity, Camera, Character, and most 2280 values are filled automatically from the preserved XML template.

Template rule

The XML keeps the template field order and Type attributes. Frontend driven values replace only their matching Value fields. All untouched fields stay as template defaults.

Optional feel controls

Feel controls are manual tuning presets. They do not come from the five frontend stat bars. They are exposed because the stock ship XML shows a few meaningful hover and recovery variations, and those variations are easier to understand as plain language options than as raw XML values.

Rebound feel

Rebound feel controls Rebound Pitch Mod. This changes how visibly the ship pitches after landing, rebounding, or bottoming out. It mostly affects the sense of weight and recovery motion, not the frontend stat bars.

  • Default rebound, 1.3: stable landing recovery with restrained nose movement. Best for most balanced ships.
  • Active rebound, 2.0: more visible nose movement after landing or bottoming out. Good for ships that should feel more reactive.
  • Aggressive rebound, 2.5: strong nose movement and dramatic recovery behavior. Best for ships that should feel lively, unstable, or more expressive.

Rebound stock examples

  • Most stock examples use the default 1.3 value, which is why 1.3 is the baseline.
  • Omnicom and Hyperion helped justify an active rebound option because their feel supports more visible movement than the safest baseline.
  • Wyvern helped justify the aggressive rebound option because it supports a stronger pitch reaction profile.

Track alignment feel

Track alignment feel controls Hover Rotation Power. This changes how strongly the ship corrects itself toward the track surface. A stronger value can make a fast or unstable ship feel more controlled, but too much correction can make the hover behavior feel stiff.

  • Standard alignment, 25: normal hover correction used by most ships. Best default choice.
  • Strong alignment, 30: stronger correction toward the track surface. Useful for very fast ships, low handling ships, or ships that need extra stability.

Track alignment stock examples

  • Most stock examples use Hover Rotation Power 25, which is why standard alignment is the default.
  • Nexus helped define the strong alignment option because it uses Hover Rotation Power 30 and shows why a faster or more demanding ship may need stronger track correction.

Why these are presets

These values are not calculated from Acceleration, Speed, Handling, Shield, or Firepower because the stock data does not show a clean frontend mapping. They are treated as optional presets so the user can choose the handling feel deliberately.

Recommended use

  • Use Default rebound and Standard alignment for a normal custom ship baseline.
  • Use Active rebound when the ship feels too flat or lifeless after landing.
  • Use Aggressive rebound only when a more dramatic recovery motion is desired.
  • Use Strong alignment when the ship feels unstable at higher speed or does not settle onto the track cleanly.

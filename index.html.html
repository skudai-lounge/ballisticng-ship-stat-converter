About

This tool converts simple frontend ship stats into BallisticNG XML stat values.

The five values you enter, Acceleration, Speed, Handling, Shield, and Firepower, are player facing summary stats. BallisticNG uses many backend XML fields to create the actual ship behavior, so one frontend value may control several XML fields.

How the rules were established

The conversion rules were rebuilt from the official Base Ship Stats sheet. Frontend stat bars were matched against the backend prefab values for the official ships.

When one frontend value had one clear backend value, the converter uses that value directly. When multiple official ships share the same frontend value but use different backend values, the converter uses an average as a balanced custom ship baseline. When no official ship exists for a frontend value, the converter uses a controlled house value and treats it as an extrapolated preset.

This means the converter is designed for predictable custom ship tuning. It should not be read as a perfect reverse engineered reconstruction of every official stock ship.

Rule types

  • Official lookup baselines. Speed, Acceleration, Handling, Shield, Firepower, and Grip are generated from lookup tables. The lookup tables are based on official ship examples.
  • Averaged official values. Some frontend values appear on multiple ships with different backend values. In those cases, the converter uses the average as a neutral custom baseline.
  • House extrapolated values. Some frontend values are not represented in the official sheet. These values remain controlled custom presets.
  • Defaults. Engine Power, most Gravity fields, Camera, Character, and most 2280 values are filled from template defaults unless exposed elsewhere.
  • Manual feel controls. Rebound Feel and Track Alignment are optional presets because they do not map cleanly to the five frontend stats.
  • Refinement profiles. Optional second pass tuning profiles are documented in their own tab.

Example conversions

  • Speed 3. Speed 3 appears on Scorpio, Protonic, and Mako with different top speed values. The converter averages those official values, so Speed 3 outputs Toxic 57.67, Apex 70.67, Halberd 85.50, Spectre 101.75, and Zen 117.17.
  • Acceleration 2. Acceleration 2 appears on G Tek, Diavolt, Caliburn, Orbitronix, and Mako. The converter averages those official values, so Acceleration 2 outputs Toxic 1.10, Apex 0.96, Halberd 0.82, Spectre 0.62, and Zen 0.45.
  • Handling 5. Handling 5 appears on Tenrai, Protonic, and MTech. The converter averages their steer and grip behavior, so Handling 5 outputs Steer Speed 1.87, Steer Gain 1.21, Steer Falloff 0.82, Steer Slide 1.25, and Grip 4.43.
  • Shield 6. Shield uses a clean official lookup. Shield 6 outputs Shield Effectiveness 0.94.
  • Firepower 7. Firepower uses a clean official lookup. Firepower 7 outputs Weapon Effectiveness 1.032.

Stock ship examples

  • Speed. Tenrai and Caliburn both use Speed 5 and share Zen 119.5. Nexus uses Speed 9 and Zen 125.0. Speed 3 varies across Scorpio, Protonic, and Mako, which is why the converter averages duplicated frontend values.
  • Acceleration. Acceleration 2 varies across G Tek, Diavolt, Caliburn, Orbitronix, and Mako. Acceleration 8 appears on Omnicom and Protonic, so the converter averages those two official examples.
  • Handling. Handling 6 appears on Wyvern, Caliburn, and Orbitronix with different steer and grip values. Handling 8 appears on G Tek and Lance and is more consistent.
  • Shield. Shield 3 maps to 0.97 across several ships. Shield 6 maps to 0.94 on Orbitronix and Mako. Shield 8 maps to 0.92 on Diavolt.
  • Firepower. Firepower 1 maps to 0.936, Firepower 5 maps to 1.000, and Firepower 8 maps to 1.048 in the official data.

Template rule

The XML keeps the template field order and Type attributes. Frontend driven values replace only their matching Value fields. All untouched fields stay as template defaults.

Optional feel controls

Feel controls are manual tuning presets. They do not come from the five frontend stat bars. They are exposed because the stock ship XML shows a few meaningful hover and recovery variations, and those variations are easier to understand as plain language options than as raw XML values.

Rebound feel

Rebound feel controls Rebound Pitch Mod. This changes how visibly the ship pitches after landing, rebounding, or bottoming out. It mostly affects the sense of weight and recovery motion, not the frontend stat bars.

  • Default rebound, 1.3: stable landing recovery with restrained nose movement. Best for most balanced ships.
  • Active rebound, 2.0: more visible nose movement after landing or bottoming out. Good for ships that should feel more reactive.
  • Aggressive rebound, 2.5: strong nose movement and dramatic recovery behavior. Best for ships that should feel lively, unstable, or more expressive.

Rebound stock examples

  • Most stock examples use the default 1.3 value, which is why 1.3 is the baseline.
  • Scorpio and Omnicom use Rebound Pitch Modifier 2.0, which defines the active rebound option.
  • Wyvern and Orbitronix use Rebound Pitch Modifier 2.5, which defines the aggressive rebound option.

Track alignment feel

Track alignment feel controls Hover Rotation Power. This changes how strongly the ship corrects itself toward the track surface. A stronger value can make a fast or unstable ship feel more controlled, but too much correction can make the hover behavior feel stiff.

  • Standard alignment, 25: normal hover correction used by most ships. Best default choice.
  • Strong alignment, 30: stronger correction toward the track surface. Useful for very fast ships, low handling ships, or ships that need extra stability.

Track alignment stock examples

  • Most stock examples use Hover Rotation Power 25, which is why standard alignment is the default.
  • Nexus helped define the strong alignment option because it uses Hover Rotation Power 30 and shows why a faster or more demanding ship may need stronger track correction.

Why these are presets

These values are not calculated from Acceleration, Speed, Handling, Shield, or Firepower because the stock data does not show a clean frontend mapping. They are treated as optional presets so the user can choose the handling feel deliberately.

Refinement profile relationship

Rebound Feel and Track Alignment are direct manual controls. Refinement Profile is a second pass that adjusts the generated backend values. A refinement profile may recommend a feel setup, but it does not automatically override the selected Rebound Feel or Track Alignment values.

Recommended use

  • Use Default rebound and Standard alignment for a normal custom ship baseline.
  • Use Active rebound when the ship feels too flat or lifeless after landing.
  • Use Aggressive rebound only when a more dramatic recovery motion is desired.
  • Use Strong alignment when the ship feels unstable at higher speed or does not settle onto the track cleanly.

Refinement profiles

Intent

The baseline converter uses official stock ship data to generate a balanced XML profile from the frontend stats. That is the safest first pass.

Using frontend stats alone is generally not advisable when trying to recreate exact backend behavior, because there are fewer frontend data points than backend tuning fields. The official sheet also shows that ships with the same frontend value can have different backend values. Frontend stats are useful summaries, but they do not fully describe a ship’s handling identity.

Refinement profiles exist to address that gap. They apply small, controlled tuning offsets after the baseline lookup has been generated. This lets the user lean the ship toward a behavior style without manually editing every XML field.

Important rule

Refinement profiles are custom tuning values. They are not official stock data and should not be treated as exact stock ship reconstruction.

The final flow is: frontend stats, baseline lookup, optional refinement profile, clamp final values, XML output.

Available profiles

  • Default baseline: no adjustment. Uses the official averaged lookup values.
  • Agile lightweight: faster response, sharper correction, and slightly stronger grip.
  • Heavy utility: slower response, calmer steering, and a weightier feel.
  • High speed specialist: higher top speed with less forgiving handling.
  • Drift focused: lower grip, more slide, and stronger airbrake identity.
  • Stable beginner friendly: more planted, more forgiving, and less dramatic.
  • Combat focused: slightly stronger combat profile without heavily changing movement.

Profile offsets

Offsets are applied after the baseline lookup values are generated.

Agile lightweight

  • Acceleration: +0.05 to Toxic, Apex, Halberd, Spectre, and Zen.
  • Steer: Steer Speed +0.05, Steer Gain +0.03, Steer Falloff +0.02, Steer Slide -0.05.
  • Grip: +0.25.
  • Airbrake Drift Falloff: +1.
  • Recommended feel: Active rebound 2.0 and Standard alignment 25.

Heavy utility

  • Acceleration: -0.05 to Toxic, Apex, Halberd, Spectre, and Zen.
  • Steer: Steer Speed -0.05, Steer Gain -0.03, Steer Falloff -0.02, Steer Slide +0.05.
  • Grip: +0.10.
  • Airbrake Drift Power: -0.02.
  • Airbrake Drift Falloff: +1.
  • Recommended feel: Default rebound 1.3. Strong alignment 30 if extra stability is needed.

High speed specialist

  • Speed: Toxic +0.50, Apex +0.75, Halberd +1.00, Spectre +1.00, Zen +1.00.
  • Acceleration: -0.05 to Toxic, Apex, Halberd, Spectre, and Zen.
  • Steer: Steer Speed -0.03, Steer Slide +0.08.
  • Grip: -0.15.
  • Airbrake Drift Power: +0.02.
  • Airbrake Drift Falloff: -1.
  • Recommended feel: Default rebound 1.3 and Strong alignment 30.

Drift focused

  • Steer: Steer Speed +0.03, Steer Gain -0.02, Steer Falloff -0.03, Steer Slide +0.12.
  • Grip: -0.40.
  • Airbrake Drift Power: +0.05.
  • Airbrake Drift Falloff: -2.
  • Recommended feel: Active rebound 2.0 and Standard alignment 25.

Stable beginner friendly

  • Speed: Toxic -0.25, Apex -0.25, Halberd -0.50, Spectre -0.50, Zen -0.50.
  • Acceleration: +0.03 to Toxic, Apex, Halberd, Spectre, and Zen.
  • Steer: Steer Speed -0.02, Steer Gain +0.02, Steer Falloff +0.03, Steer Slide -0.08.
  • Grip: +0.35.
  • Airbrake Drift Power: -0.03.
  • Airbrake Drift Falloff: +2.
  • Recommended feel: Default rebound 1.3 and Strong alignment 30.

Combat focused

  • Speed: Zen -0.25 only.
  • Shield Effectiveness: -0.01. Lower value means stronger shield.
  • Weapon Effectiveness: +0.016.
  • Firepower 0 remains -1.000 and does not receive the weapon offset.

Clamp rules

  • Speed does not go below Speed 1 values or above Speed 10 extrapolated values.
  • Acceleration does not go below Acceleration 1 values or above Acceleration 10 values.
  • Steer Speed stays between 1.50 and 2.05.
  • Steer Gain stays between 1.05 and 1.40.
  • Steer Falloff stays between 0.65 and 1.00.
  • Steer Slide stays between 0.80 and 1.60.
  • Grip stays between 3.30 and 8.00.
  • Airbrake Drift Power stays between 0.80 and 0.95.
  • Airbrake Drift Falloff stays between 3 and 14.
  • Shield Effectiveness stays between 0.90 and 0.99.
  • Weapon Effectiveness stays between -1.000 and 1.080.
